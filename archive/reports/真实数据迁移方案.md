> 创建：2026年6月5日 | 目标：将所有模拟数据替换为江苏省教育考试院官方真实数据

---

| 表名 | 行数 | 模拟程度 | 问题 |

## 二、逐模块迁移方案

### 模块 1：admission_scores（录取分数线）🔴 优先级最高

**现状**：7,638条，模拟生成。`generate_realistic_scores()` 用 BASE_SCORES + 随机噪声生成。

**真实数据来源**：

| 来源 | URL | 格式 | 难度 |
|------|-----|------|------|
| 江苏省教育考试院 | http://www.jseea.cn/ | PDF 公告 | 高（需PDF解析） |
| 教育部阳光高考 | https://gaokao.chsi.com.cn/ | 网页表格 | 中（需爬虫） |
| 各高校招生官网 | 各校 .edu.cn | 网页/Excel | 中（需逐个爬取） |
| **中国教育在线** | https://www.eol.cn/ | 结构化网页 | **低（推荐）** |
| **掌上高考/夸克** | 第三方App | API（非公开） | 高（有反爬） |

**推荐方案**：使用中国教育在线(eol.cn)的院校录取分数线页面。该网站对每所院校提供历年分数线表格，HTML 结构规范，易于解析。

**数据字段映射**：

```
真实字段                → 数据库字段
─────────────────────────────────────
院校名称               → universities.name (匹配 university_id)
专业名称               → majors.name (匹配 major_id)
年份                   → year
科类(物理/历史)        → subject_type
批次(本科批)           → batch
最低录取分             → min_score
最低录取位次           → min_rank
平均分                 → avg_score
招生计划数             → plan_count
```

**表结构**：✅ 无需修改。当前结构完全兼容真实数据。

**导入脚本设计**（`import_real_admission.py`）：

```
Phase 1: 从 eol.cn 爬取数据
  1. 遍历 universities 表获取55所院校名称
  2. 构造 URL: https://www.eol.cn/jiangsu/XXXXX/
  3. 解析 HTML table，提取历年录取线
  4. 输出到 real_admission.csv

Phase 2: 数据清洗
  1. 院校名称模糊匹配 → university_id
  2. 专业名称模糊匹配 → major_id（无匹配时=NULL）
  3. 位次标准化（去逗号/空格）
  4. 年份/科类/批次标准化

Phase 3: 导入
  1. 备份当前 admission_scores
  2. DELETE 旧数据 / 或 UPSERT
  3. LOAD DATA 新 CSV
  4. 验证：每校每年每科类至少1条

风险评估: 🟡 中
  - eol.cn 可能有 IP 反爬限制
  - 院校/专业名称匹配可能不准确
  - 部分院校/专业可能无数据
  缓解: 分批次爬取 + 人工审核匹配结果
```

### 模块 2：score_segments（一分一段表）🔴 优先级最高

**现状**：162条，每5分采样（300-690），仅物理类，无历史类。`seed.py` 生成。

**真实数据来源**：

| 来源 | URL | 格式 |
|------|-----|------|
| 江苏省教育考试院 | http://www.jseea.cn/ | PDF（每年一份） |
| 各高考资讯站 | 百家号/搜狐等 | 网页文章（整理版） |

**推荐方案**：一分一段表每年只有一份 PDF（物理类+历史类各一张表），手动下载后用 pdfplumber 解析，或直接使用教育自媒体发布的整理版数据（结构化CSV，准确性需验证）。

**数据字段**：

```
真实字段          → 数据库字段
─────────────────────────────
分数              → score
全省位次          → rank
科类(物理/历史)   → subject_type
年份              → year
省份ID            → province_id
```

**表结构**：✅ 无需修改。

**关键差异**：真实一分一段表每个分数都有对应位次（如 2024 江苏物理类 300-690 共 391 行），当前仅 54 行。须全量替换，保留现有的 `_interpolate_rank()` 插值逻辑作为降级策略。

**导入脚本设计**（`import_real_segments.py`）：

```
Phase 1: 获取数据
  方式A: 手动下载 PDF → pdfplumber 提取表格 → CSV
  方式B: 从教育资讯网站复制整理版 → CSV

Phase 2: 数据清洗
  1. 确保每个分数(300-690)都有一条记录
  2. 位次去逗号、转整数
  3. 验证：位次随分数单调递减

Phase 3: 导入
  1. DELETE FROM score_segments
  2. INSERT 真实数据
  3. 验证：线性插值 610-615 连续无跳变

风险评估: 🟢 低
  - 一分一段表是官方公开发布的确定数据
  - 每年只有两份（物理+历史），数据量小
  - 手动录入也可行（391行×3年=1173行）
```

### 模块 3：major_profiles（专业就业画像）🟡 优先级中

**现状**：13条人工编写数据。覆盖13/30专业（计算机/AI/软件等前13个）。

**真实数据来源**：

| 来源 | 内容 | 获取方式 |
|------|------|----------|
| 教育部就业报告 | 各专业就业率/薪资 | 公开下载 PDF |
| 麦可思《中国大学生就业报告》 | 专业薪资排名 | 购买报告/公开摘要 |
| 各高校就业质量报告 | 分专业就业率/去向 | 官网公开下载 |
| 智联/前程无忧招聘数据 | 行业薪资水平 | 公开报告 |

**表结构**：✅ 无需修改。

**数据字段**：

```
avg_starting_salary:  起薪（来自麦可思/就业报告）
employment_rate_3yr:  毕业三年后就业率
top_industries:       主要就业行业（JSON数组）
top_positions:        主要就业岗位（JSON数组）
top_cities:           主要就业城市（JSON数组）
postgrad_rate:        读研深造比例
holland_code:         霍兰德职业代码（可计算）
data_year:            数据年份
```

**导入脚本设计**：

```
Phase 1: 收集公开数据
  1. 下载《中国大学生就业报告》公开摘要
  2. 整理各专业薪资/就业率数据为 Excel
  3. 输出到 real_profiles.csv

Phase 2: 匹配
  按专业名称匹配 → major_id
  
Phase 3: 导入
  DELETE + INSERT，保持当前13条+补充17条

风险评估: 🟡 中
  - 第三方报告数据可能有版权限制
  - 公开摘要只有 Top 10/20 专业，全部30个专业的数据不全
  缓解: 标注数据来源，未覆盖专业显示"暂无数据"
```

## 三、数据更新流程

### 每年出分后（6月24日左右）

```
第1天：获取当年一分一段表
  → 从教育考试院下载 PDF
  → 解析/手动录入 → CSV
  → 运行 import_real_segments.py

第2-3天：获取当年录取分数线
  → 爬取/手动收集各校各专业录取分
  → 清洗匹配 → CSV
  → 运行 import_real_admission.py

第4天：更新就业数据（如有新报告）
  → 运行 import_real_profiles.py

第5天：验证
  → 运行回归测试（10 场景）
  → 抽查 5 所院校 × 5 个专业
```

### 自动化程度

```
一分一段表:   手动为主（每年仅2张表，391行×2）
录取分数线:   半自动（爬虫+人工审核）
就业数据:     手动为主（每年更新1次）
```

## 四、风险评估

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| 爬虫被反爬/IP封禁 | 🟡 中 | 分批次、加延迟、使用代理 |
| 院校/专业名称无法精确匹配 | 🟡 中 | 人工审核匹配结果 |
| 第三方数据版权问题 | 🟡 中 | 标注数据来源，避免批量复制商业数据库 |
| 真实数据与用户预期不符 | 🟢 低 | 页面标注"数据来源：江苏省教育考试院" |
| 导入脚本覆盖线上数据 | 🔴 高 | 导入前先备份，先在测试库验证 |
| 部分院校/专业无公开数据 | 🟡 中 | 标注"暂无数据"，不让页面报错 |
